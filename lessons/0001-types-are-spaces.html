Types are Spaces, Equality is a Path

The one idea the whole subject grows from — and why it forces you to give up something you’ve never questioned about equality.

You came here to understand foundations: what “types as spaces” really means, and why anyone would rebuild mathematics on it. This first lesson plants the seed. By the end you should be able to read the expression \(p : x =_A y\) out loud, correctly, and explain why it can have more than one inhabitant — the single fact that separates this world from the set-theoretic one you already know.

The central idea “The central new idea in homotopy type theory is that types can be regarded as spaces in homotopy theory, or higher-dimensional groupoids in category theory.”[1, ch.2]

The dictionary

You already know the right-hand column. The work of this course is to earn the left-hand column — to see that the type-theoretic objects on the left genuinely behave like the homotopical objects on the right. Start by memorizing the correspondence; we will justify each row over the coming lessons.

Type theoryHomotopy theoryYou read it as…
type \(A\)a space“the space \(A\)”
term \(a : A\)a point of \(A\)“\(a\) is a point of \(A\)”
identity type \(a =_A b\)the path space from \(a\) to \(b\)“the type of paths from \(a\) to \(b\)”
term \(p : a =_A b\)a path from \(a\) to \(b\)“\(p\) is a path from \(a\) to \(b\)”
\(\refl{a} : a =_A a\)the constant path at \(a\)“\(a\) equals itself, reflexively”
\(r : p =_{(a =_A b)} q\)a homotopy between paths“a path between paths \(p\) and \(q\)”

Why two equalities? \(a \jdeq b\) (judgmental) is true by computation — like \(2+2 \jdeq 4\). It is invisible, carries no data, and lives in the meta-theory. \(a =_A b\) (propositional) is a type you can inhabit, transport along, and interrogate. The gap between them is where homotopy lives.

Equality becomes a type — and that changes everything

In set theory, “\(a = b\)” is a proposition: it is either true or false, full stop. In type theory, equality is given by a type, the identity type \(a =_A b\), and its terms are the proofs.[1, §1.12] There is exactly one canonical way to build such a proof when something is equal to itself — reflexivity, \(\refl{a} : a =_A a\), the constant path.

Here is the move that breaks set-level intuition. Once equality is a type, nothing forces it to have at most one inhabitant:

From the book “Nothing prevents the type \(a = b\) from … including more information. Indeed, this is the cornerstone of the homotopical interpretation, where we regard witnesses of \(a = b\) as paths … Just as there can be more than one path between two points of a space, there can be more than one witness that two objects are equal.”[1, §1.12]

This is proof-relevance. A proof of equality is data, and two proofs that \(a = b\) need not themselves be equal. For a topologist this should feel familiar and welcome: the loops at a basepoint of the circle are all “paths from \(x\) to \(x\),” but they are genuinely different — they wind different numbers of times. The identity type \(x =_{S^1} x\) will turn out to be \(\mathbb{Z}\). Equality, made into a type, remembers the homotopy.

The tower, for free

Because \(a =_A b\) is itself a type, you can ask the same question one level up: given two parallel paths \(p, q : a =_A b\), what is \(p =_{(a =_A b)} q\)? That is a type of paths between paths — a homotopy. And then paths between those. Iterating the identity type builds an infinite tower[1, ch.2]:

points \(\;\rightsquigarrow\;\) paths \(\;\rightsquigarrow\;\) paths-between-paths \(\;\rightsquigarrow\;\) \(\cdots\)

This tower, with its composition, inverses, and identity at every level, is exactly the structure of a (weak) \(\infty\)-groupoid — the same structure a topological space carries through its points and higher paths. Every type is one, automatically. And the engine that generates the entire structure is a single rule, path induction, which we meet in Lesson 2.

Synthetic, not analytic. Classical homotopy theory is analytic: a path is built out of points, \(p : [0,1] \to X\). HoTT is synthetic: points, paths, and higher paths are primitive, indivisible notions, governed by rules rather than assembled from a substrate.[1, ch.2] This is the foundational payoff — and the reason the theory can serve as a foundation at all.

Check yourself

Answer from memory before revealing — retrieval is what makes this stick.

1. How do you read \(p : x =_A y\) under the homotopy interpretation?

  1. \(p\) is a point of the path from the space
  2. \(p\) is a path from point \(x\) to \(y\)
  3. \(x\) is a path from the point \(y\) to \(p\)
  4. \(A\) is a path from the point \(x\) to \(y\)

\(A\) is the space, \(x\) and \(y\) are points of it, and \(p\) — a term of the identity type — is a path from \(x\) to \(y\).

2. What is the deepest break from set-theoretic equality?

  1. Equality of any two terms is always false
  2. Equality holds only when terms compute
  3. An equality proof is data, possibly many
  4. Equality is a proposition that is decidable

Proof-relevance: \(a =_A b\) is a type whose inhabitants are proofs, and there may be many distinct ones — “more than one path between two points.” In set theory equality is a mere true/false proposition.

3. Which pair names the two different equalities, in order?

  1. judgmental \(a \jdeq b\); propositional \(a =_A b\)
  2. structural \(a \jdeq b\); numerical \(a =_A b\)
  3. canonical \(a \jdeq b\); reflexive \(a =_A b\)
  4. decidable \(a \jdeq b\); computable \(a =_A b\)

\(a \jdeq b\) is judgmental (definitional) equality — true by computation, invisible, no data. \(a =_A b\) is propositional equality — a type whose terms are proofs.

4. Why does every type carry \(\infty\)-groupoid structure?

  1. Because every type is finite and is fully discrete
  2. Because each type is, by an axiom, a true group
  3. Because identity types iterate into a path tower
  4. Because spaces are assumed as a true substrate

\(a =_A b\) is itself a type, so you can form identity types of identity types, ad infinitum. That tower of (higher) paths, with composition/inverse/identity at each level, is precisely a weak \(\infty\)-groupoid.
