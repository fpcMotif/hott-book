Path Induction: the One Engine

In Lesson 1 you saw that every type is an \(\infty\)-groupoid — points, paths, paths-between-paths, with inverses and composition at every level. Here is the surprise: none of that structure is assumed. It is all generated by a single rule.

For a foundations-minded reader this is the crux. In ordinary mathematics you would axiomatize a groupoid: postulate composition, postulate inverses, postulate associativity. In HoTT you postulate none of it. You write down one elimination rule for the identity type — path induction, traditionally \(\J\) — and inverses, concatenation, and every groupoid law fall out as theorems.[1, §2.1]

The rule

Recall the identity type comes with exactly one constructor, \(\refl{x} : x =_A x\). Path induction is its elimination rule. Stated as the book does[1, §1.12]: to define something depending on a path, it is enough to define it on reflexivity.

Path induction (J) Given a family over the identity type \[ C : \textstyle\prod_{x,y:A}\,(x =_A y)\to\UU \] to construct \(f(x,y,p) : C(x,y,p)\) for all \(x,y:A\) and all \(p : x =_A y\), it suffices to give an element \[ c : \textstyle\prod_{x:A} C(x,x,\refl{x}). \] Path induction then yields \(f\), with the computation rule \[ f(x,x,\refl{x}) \;\jdeq\; c(x). \]

Same shape as every induction you know. \(\mathbb{N}\)-induction: prove it for \(0\) and for \(\mathsf{succ}\). Coproduct: prove it for \(\mathsf{inl}\) and \(\mathsf{inr}\). Identity type: prove it for \(\refl{}\) — the single generator. The computation rule is the analogue of “the \(0^{\text{th}}\) term really is the one you specified.”[1, §1.12]

The working slogan, which you will use constantly:

“By induction, it suffices to assume \(y\) is \(x\) and \(p\) is \(\refl{x}\).”[1, §2.1] Every proof about a general path begins this way, collapsing the path to the constant one.

Watch it generate structure: path inversion

Symmetry of equality — “paths can be reversed” — is Lemma 2.1.1 in the book.[1] We want, for every \(p : x =_A y\), an inverse \(\opp{p} : y =_A x\). Here is the entire proof, in the informal style:

Proof (Lemma 2.1.1) We must construct \(\opp{p} : y =_A x\) for all \(p : x =_A y\). By induction, it suffices to assume \(y\) is \(x\) and \(p\) is \(\refl{x}\). Then the goal type \(y =_A x\) is just \(x =_A x\) — and we have an inhabitant ready to hand: \(\refl{x}\). So set \(\opp{\refl{x}} \defeq \refl{x}\), and J extends this to all paths. ∎

Read what happened. The reflexivity case forced the two endpoints together, so the type we had to inhabit (\(y=x\)) and a type we already had (\(x=x\)) coincided. That collapse is the whole trick — and it is why the computation rule reads \(\opp{\refl{x}} \jdeq \refl{x}\): on constant paths, reversal does nothing, exactly as in topology.

The same move, applied twice (induct on \(p\), then on \(q\)), gives concatenation \(\ct\): from \(p : x=_A y\) and \(q : y=_A z\), a path \(p \ct q : x=_A z\), with \(\refl{x} \ct \refl{x} \jdeq \refl{x}\).[1, Lemma 2.1.2] Iterate the idea and you get the unit, inverse, and associativity laws too — the groupoid structure, derived, never assumed.

Your turn: prove concatenation. Try it, then reveal.
Goal: build \(p \ct q : x =_A z\) from \(p : x=_A y,\ q : y=_A z\). By induction on \(p\), assume \(y\) is \(x\) and \(p\) is \(\refl{x}\); the goal becomes \(x =_A z\) and \(q\) now has type \(x =_A z\). By induction on \(q\), assume \(z\) is \(x\) and \(q\) is \(\refl{x}\); the goal is \(x =_A x\). Provide \(\refl{x}\). ∎  (You could have stopped after the first induction and returned \(q\) — a different term of the same type. Proof-relevance means the choice matters: it changes the computation rule.[1, §2.1])

The puzzle from Lesson 1, resolved

Lesson 1 left a paradox hanging. Path induction seems to say “every path is \(\refl{}\)” — so how can \(x =_{S^1} x\) have many distinct elements? The resolution is precise and worth holding onto:[1, §1.12, Rmk 1.12.3]

It is the family, not the type Path induction does not say each identity type is generated by \(\refl{}\). It says the identity family \(\big(x =_A y\big)\), with \(x,y\) both varying, is generated by the diagonal elements \(\refl{x}\). You may collapse \(p\) to \(\refl{}\) only when you are simultaneously free to move both endpoints. For a loop type \(x =_A x\) with the endpoints fixed and equal, J does not apply — so it cannot, and does not, prove that every loop is \(\refl{}\).[1]

Check yourself

Recall first, then reveal.

1. Path induction lets you define \(f(p)\) for all \(p : x =_A y\) by specifying its value only in which case?

  1. when the path \(p\) is any chosen fixed loop
  2. when the path \(p\) is \(\refl{}\) and endpoints coincide
  3. when the type \(A\) is known to be a set
  4. when both endpoints are kept distinct and fixed

J reduces the general case to the single reflexivity case \((x,x,\refl{x})\): both endpoints become the same \(x\) and the path becomes \(\refl{x}\). The computation rule then says \(f(\refl{x}) \jdeq c(x)\).

2. Building \(\opp{p} : y=_A x\) from \(p : x=_A y\) by J, the reflexivity case asks for…

  1. an element of \(x=_A x\), provided by \(\refl{x}\)
  2. an element of \(y=_A x\), provided by \(\opp{p}\)
  3. an element of \(x=_A y\), provided by the \(p\)
  4. an element of \(y=_A y\), provided by \(\refl{y}\)

In the reflexivity case \(y\) becomes \(x\), so the goal \(y=_A x\) collapses to \(x=_A x\), which \(\refl{x}\) inhabits. Hence \(\opp{\refl{x}} \jdeq \refl{x}\).

3. Why can’t path induction prove “every loop \(p : x=_A x\) equals \(\refl{x}\)”?

  1. Because \(\refl{x}\) is not really a loop at the point
  2. Because identity types are by assumption always empty
  3. Because J generalizes the family, needing both endpoints free
  4. Because every type is a set with one path

J generates the identity family (\(x,y\) both varying) from the diagonal \(\refl{x}\). A loop type fixes both endpoints equal, so the reduction is unavailable — and indeed loops like those in \(x=_{S^1}x\) need not be \(\refl{}\).

4. In HoTT, the groupoid laws (inverse, composite, associativity) are…

  1. postulated as separate axioms on each given type
  2. true only for types that happen to be sets
  3. derived as theorems from one rule, path induction
  4. imported from classical topology by a translation

This is the foundational payoff: a single elimination rule for \(=\) generates inversion, concatenation, and all higher coherence laws. Nothing about groupoids is assumed.
